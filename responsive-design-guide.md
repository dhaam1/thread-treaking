# Tailwind CSS 기반 교과서적 반응형 가이드

본 문서는 훌륭한 PC 및 모바일 최적화 기법을 정리한 가이드입니다. 다른 웹 프로젝트 진행 시에도 그대로 가져다가 적용할 수 있도록 핵심 원칙들을 교과서적으로 정리하였습니다.

---

## 1. 데스크톱(PC) 최적화 교과서적 패턴

모바일뿐만 아니라 폭이 아주 넓은 데스크톱 모니터에서도 안정적인 UI를 제공하기 위한 기법들입니다.

*   **최대 너비(Max-Width) 제한 및 중앙 정렬 (`mx-auto`)**:
    PC 해상도가 아무리 커지더라도 글씨나 이미지가 무한정 양옆으로 찢어지지 않도록, 특정 너비 이상(예: `max-w-[1632px]`, `maxWidth: '1472px'`) 커지지 않게 제한을 두고 `mx-auto`를 설정해 화면 정중앙에 컨테이너를 가두는 것이 기본입니다.
*   **풍부한 마우스 인터랙션 (Hover & Transition)**:
    PC는 터치가 아닌 마우스 포인터가 있으므로, 요소에 마우스를 올렸을 때(`hover:`)의 반응을 적극 활용해야 합니다. `transition-all duration-300`, `hover:opacity-80`, `group-hover:w-full` 등을 활용해 버튼 클릭이나 메뉴 진입 시 우아한 밑줄이나 색상 트랜지션 효과를 줍니다.
*   **Sticky 네비게이션 및 다단 메뉴 (Mega Menu)**:
    상단 헤더 공간이 넓으므로 `sticky top-0 z-20`, `backdrop-blur` 속성을 사용해 네비게이션을 고정해 두고, 메뉴 호버 시 자식 카테고리가 팝업처럼 떨어지는 다단 드롭다운 형태(Mega Menu)를 구성하는 것이 데스크톱 PC 전용 레이아웃의 필수 문법입니다.

## 2. 데스크톱/모바일 다단 레이아웃(DOM) 완전 분리

Header나 Footer처럼 모바일과 데스크톱의 배치 순서나 구조가 완전히 다른 경우, 억지로 하나의 HTML(박스)로 맞추려 하지 않습니다.

*   **구조적 분기 렌더링**:
    단순 폰트/너비 변경이 아니라 구조 자체가 바뀌어야 할 경우, PC용 뼈대(`<div className="hidden lg:flex">`)와 모바일용 뼈대(`<div className="flex flex-col lg:hidden">`) 두 개를 완전히 분리해서 만들어 둡니다. 반응형 깨짐 오류를 가장 확실하고 깔끔하게 막아주는 교과서적인 방법입니다.

## 3. 고정 픽셀(px) 배제 및 상대적 너비(`w-full`) 활용

어떠한 기기 폭(핸드폰 기종)에서도 레이아웃이 깨지지 않도록 기본 너비를 화면 비율에 맞게 유동적으로 설정해야 합니다. 컨테이너의 크기를 고정시키는 것은 반응형 레이아웃의 가장 큰 적입니다.

*   **기본 컨테이너 설정**: 박스에 `w-[300px]` 등의 고정값을 쓰지 않고 `w-full` (가로 100%)을 기본으로 설정합니다. 가로로 꽉 차게 만들되, 최대 폭이 필요한 경우 `max-w-screen-md`나, 설정 파일에 등록한 `max-w-content` 등을 조합해 제한을 둡니다.
*   **꽉 채우기 (Inset)**: 이미지나 영상이 어떤 화면이든 부모 박스를 여백 없이 꽉 채우도록 `absolute inset-0 z-0 overflow-hidden` 패턴을 사용합니다. (`inset-0`은 `top-0 right-0 bottom-0 left-0`과 동일한 효과입니다.)

## 4. 모바일 전용 UI 패턴 (오버레이 및 스크롤 잠금)

모바일 화면의 부족한 공간을 극복하기 위해 가장 중요한 교과서적 기술 중 하나입니다.

*   **풀스크린 오버레이 네비게이션**:
    PC에서는 상단에 메뉴를 길게 늘어놓지만, 모바일에서는 햄버거 버튼(☰)을 통해 화면 전체를 덮는(`fixed inset-0 w-full h-screen-safe z-50`) 네비게이션 메뉴를 띄우는 것이 좋습니다.
*   **배경 스크롤 잠금 (Scroll Lock)**:
    모바일 풀스크린 메뉴가 열렸을 때 사용자가 스와이프를 하면 뒤쪽의 본문이 같이 스크롤되는 현상을 막기 위해, 메뉴 Open 시 `document.body.style.position = 'fixed'` 처리를 해주는 것은 모바일 UX 반응형 대응의 필수 디테일입니다.

## 5. 이미지/비디오 비율 관리 및 기기별 에셋 스플리팅

모바일 기기의 너비가 변하더라도 미디어가 찌그러지거나 잘리는 현상을 방지하고 심미성을 유지해야 합니다. 

*   **비율 유지 및 채우기 (`object-cover`)**: `<Image className="object-cover" fill />` 패턴을 적용하면 화면 너비 대 높이 비율이 달라지더라도, 찌그러지지 않고 썸네일처럼 예쁘게 잘려 보여집니다.
*   **모바일용 / 데스크톱용 미디어 분리 로드**: 
    데스크톱에서의 가로형 동영상/이미지 소스와 모바일에서의 세로형 소스를 각기 다른 해상도로 준비하여, 화면 폭에 맞게 다른 미디어를 렌더링시킵니다.
    ```tsx
    // 데스크톱 최적화 가로 영상 (모바일에서는 숨김)
    <video className="hidden video-md:block absolute ... min-w-full min-h-full" ... />
    
    // 모바일 최적화 세로 영상 (데스크톱에서는 숨김)
    <video className="block video-md:hidden absolute ... min-w-full min-h-full" ... />
    ```

## 6. Flex를 이용한 유연한 레이아웃 구성

디바이스의 폭이 달라지면 콘텐츠가 차지할 수 있는 폭도 달라집니다. 어떤 기기 너비에서도 요소가 항상 화면의 정중앙에 위치하도록 배치합니다.

*   **절대 좌표 + 중앙 정렬 매직 포뮬러 (`left-1/2 -translate-x-1/2`)**: 
    텍스트 그룹이나 특정 요소를 화면 가운데로 정렬 시, `absolute left-1/2 -translate-x-1/2` 를 줍니다. 이렇게 하면 화면 너비가 375px이든 430px이든 언제나 정가운데를 스스로 찾아갑니다.
*   **Flex Box 구성**: 
    요소 내부에선 `flex flex-col items-center`를 적용해 내부 글씨들도 자동으로 가운데 정렬되도록 하여 깔끔한 레이아웃을 보장합니다.

## 7. Breakpoint 기반 폰트 사이즈 및 여백 제어

무작정 비율만 줄이는게 아니라, 각 환경(모바일, 태블릿, PC)에 심미적으로 어울리는 별도의 폰트 크기, 줄바꿈, 섹션 높이를 선언해야 합니다. Tailwind에서는 주로 `md:` (768px), `lg:` (1024px) 단위로 나눕니다.

*   **기기별 제목 크기 및 뷰 분리**:
    같은 텍스트라도 모바일 화면에는 모바일에 어울리는 크기(`16px`)와 작은 화면용 줄바꿈 텍스트 구조로 두고, PC는 PC에 어울리는 크기(`20px`)와 넓은 화면용 줄바꿈 구조로 디자인을 나눕니다.
    ```tsx
    // 모바일용 텍스트 (줄바꿈이 다름)
    <h1 className="block md:hidden" style={{ fontSize: "16px" }}>...</h1>
    
    // PC/태블릿용 텍스트 (줄바꿈이 다름)
    <h1 className="hidden md:block" style={{ fontSize: "20px" }}>...</h1>
    ```
*   **뷰포트 대비 높이 (`svh`, `vh`) 조절**:
    섹션의 높이를 모바일에서는 `h-[240svh]`, PC에서는 `md:h-[320svh]` 등 독립적으로 줘서, 높이 역시 화면 환경에 가장 우아하고 아름다운 길이로 분리하여 대응합니다. (`svh`는 주소창이 숨겨지는 모바일 브라우저의 화면 변화까지 고려한 높이 단위입니다)

> **💡 핵심 요약**
> 데스크톱의 여유 공간을 100% 살리는 특화 패턴부터 모바일의 좁은 화면을 극복하는 레이아웃 분리와 오버레이 패턴까지, 고정값 배제와 Flex 기법을 철저히 지키면 어떤 화면에서도 완벽하게 유지되는 진정한 풀-반응형 웹을 구현할 수 있습니다.
